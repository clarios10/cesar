Journey to Azure — Visión Ejecutiva

Meca | Migración a nube Microsoft Azure · 4 etapas · ~9 meses estimados

4–6 sem 6–8 sem 12–16 sem Continuo
Etapa 1
Assessment & Planeación
⏱ 4 – 6 semanas
🔍 Levantamiento técnico
Inventario de VMs, SOs, versiones, dependencias. Identificación de servidores delicados (APPS-2) y críticos (GSS, SQL).
💰 TCO Comparativo
Costo total on-premises vs. Azure. Incluye hardware, licencias, energía, personal y contratos Veeam.
⚠️ Análisis de riesgos
Compatibilidad de GSS01 con Azure, licencias SQL, estado de APPS-2 y dependencias de ID de hardware.
🗺️ Roadmap por fases
Secuencia de migración ordenada por criticidad, riesgo y dependencias entre servicios.
Resultado clave
Documento de Assessment + Plan ejecutivo aprobado
Go / No-Go de migración
Etapa 2
Fundación Cloud (Landing Zone)
⏱ 6 – 8 semanas
🏗️ Suscripción & Tenant
Creación de suscripciones Azure, Management Groups, políticas de gobernanza y etiquetado.
🌐 Red & Conectividad
VNet Hub-Spoke, subnets segmentadas, VPN Site-to-Site a Ubicación #1 y #2, NSG y firewall Azure.
🪪 Identidad híbrida
Configuración de Entra ID (AAD), sincronización con DC on-premises vía EntraConnect, MFA y RBAC.
💾 Backup & Storage
Recovery Services Vault, Azure Backup para los 3 servidores prioritarios, políticas de retención.
Resultado clave
Landing Zone certificada lista para producción
Ambiente Azure productivo
Etapa 3
Migración por Fases
⏱ 12 – 16 semanas
🟢 Fase A — Bajo riesgo
APPS-3 (diseño 3D), DC secundario (DC-5). Lift & shift sin dependencias críticas. Validación de conectividad.
🔵 Fase B — DRP activo
Azure Site Recovery para servidores core. Replicación continua, prueba de failover y validación de RTO/RPO.
🟡 Fase C — Core de negocio
GSS01 (ERP Manufactura), SQL Server, APPS de producción, DC-3. Migración con ventana de mantenimiento.
🔴 Fase D — Servidor delicado
APPS-2 con plan de troubleshooting, rollback documentado y pruebas extensivas de IDs y aplicaciones.
Resultado clave
VMs migradas a Azure + DRP operativo en todas las cargas
Failover probado y funcional
Etapa 4
Optimización & Operación
⏱ Continuo post-migración
💹 FinOps & Costos
Reserved Instances, Azure Hybrid Benefit (SQL + Windows), rightsizing de VMs, alertas de presupuesto.
🛡️ Seguridad avanzada
Microsoft Defender for Cloud, Microsoft Sentinel (SIEM), revisión de políticas y accesos privilegiados.
⚙️ Modernización PaaS
Evaluación de SQL Managed Instance, App Service para IIS, posible containerización de cargas.
📊 Monitoreo & Observabilidad
Azure Monitor, Log Analytics, dashboards ejecutivos de disponibilidad, performance y costos.
Resultado clave
Plataforma madura, documentada y optimizada en costo
Operación cloud estable
